# TeamFlow — Estado de fases

> SaaS de gestión de proyectos (Kanban tipo Linear/Height) con **organizaciones multi-tenant**. Leer al iniciar sesión de trabajo.
> Vara de calidad: **FinPulse** (NO NEGOCIABLE — ver PORTFOLIO_PROJECTS.md §2 "Quality gate"), incluida la **Capa de Demo Guiada** completa.
> **Deploy diferido**: se construye completo menos el deploy en la nube; los deploys se hacen en bloque al final (decisión del usuario 2026-06-18).

## Stack
- **Front:** Angular 20 (standalone + signals) + Tailwind v4 + Angular CDK (drag&drop). **Modo oscuro de primera clase** + claro. Atajos de teclado visibles (kbd chips).
- **Back:** .NET 9 Web API (Clean Architecture) — **multi-tenancy por organización (workspace)**, aislamiento de filas por `WorkspaceId` con query filter global.
- **DB:** SQL Server 2022 (local, Windows auth).
- **Auth:** JWT + refresh rotativo + lockout · **RBAC por workspace** (Owner/Admin/Member/Viewer) · invitaciones por email. OAuth2 (Auth0/Google) documentado como opción de producción (extension point); el demo usa email/contraseña self-hosted.
- **Deploy:** Azure (diferido).

## Cómo correr (dev)
1. SQL Server local (Windows auth) en `localhost`. La API crea/migra/seedea la BD `TeamFlow` al arrancar.
2. `cd backend && dotnet run --project src/TeamFlow.Api` → API en `http://localhost:5190` (`/health`).
3. `cd frontend && npx ng serve` → app en `http://localhost:4200` (proxy directo a `:5190` vía `environment.apiBase`). Login con cualquier cuenta demo.
